# 开发指引

## 用途

存放开发过程中需要遵循的工作流、规范、约定。

包括：

- 编码规范与风格指南
- 开发工作流
- Git 提交规范
- AI Agent 协作规范（Agent 如何读文档、如何提问、如何提交代码）
- Prisma 使用规范
- 测试规范
- 代码审查检查清单

## 与 AGENTS.md 的关系

`../AGENTS.md`（架构宪法）定义「不能做什么、必须做什么」。

本目录的文档定义「怎么做」。

## 命名规范

```
kebab-case-topic.md
```

### 示例

```
coding-standards.md
agent-collaboration-guide.md
git-workflow.md
prisma-usage.md
code-review-checklist.md
```
