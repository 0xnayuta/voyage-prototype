# Phase 1: MVP — 核心循环

---

## 目标

实现航海贸易经营的核心循环：**交易 → 航行 → 交易 → 升级**。

## 必须实现

### 港口系统
- 港口定义（名称、区域、描述）
- 港口特产和价格系数
- 港口间距离配置

### 商品系统
- 商品定义（名称、分类、基础价格、单位体积）
- 商品和港口的关联（每个港口的买入/卖出价不同）

### 买卖系统
- 在港口查看商品列表和价格
- 买入商品（扣金币、增加货舱、加货物）
- 卖出商品（加金币、减货物）
- 货舱容量管理
- 利润计算

### 航行系统
- 查看可前往港口列表
- 选择目的港
- 航行耗时计算（距离 / 船速）
- 抵达逻辑
- 航行中随机事件触发（事件内容本身 Phase 2 丰富）

### 价格波动系统
- 不同港口同一商品不同价
- 买入/卖出影响该港口该商品价格
- 市场均值回归

### SQLite 存档
- Save + JSON 列模式
- 自动存档（每次操作后写入）
- 读档恢复

## 暂不实现

- 战斗系统
- 公会系统
- 排行榜
- 聊天系统
- 联机功能
- 成就系统
- 图鉴系统
- 剧情系统
- MOD 支持
- 多个手动存档位（只做自动存档）
- 图形地图（只用文字列表）
- 音效和音乐
- 动画

## 开发原则

1. 先完成核心循环（买 → 卖 → 航 → 升级），再加入更多内容
2. 所有数值配置化（`src/data/`），方便调整平衡性
3. 避免过度设计——只做当前必要的，不为「以后可能需要」提前做
4. 每完成一个子系统，写测试覆盖关键路径

## 完成标准

- 可以完成至少一轮完整的「买 → 航 → 卖」循环
- 可以退出游戏后重新加载，从存档恢复
- 至少 3 个港口、5 种商品、2 种船只
